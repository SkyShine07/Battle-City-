# 游戏概述

坦克大战是一个经典的俯视角、双人合作、防守反击的射击策略游戏。该游戏采用简单的俯视角，玩家操控一辆小坦克，在由砖墙、钢墙、森林和水域组成的固定场景中与地方坦克战斗。

# 核心玩法与目标

 由玩家(1人或者多人）保护屏幕下方的**鹰状基地**，并在所有玩家生命耗尽前(玩家初始拥有3条），消灭所有敌方坦克。

# 玩家能力

- 方向键控制坦克的移动(上下左右)
- A/B键发射子弹，子弹在碰到障碍物或者屏幕边缘消失。

## 2. 道具生成与交互机制补充

### 2.1 生成规则

1. **触发源头**：仅当**红色闪光的目标坦克**被玩家摧毁时才会生成道具，普通敌方坦克不会掉落道具
2. **生成位置**：在被摧毁的目标坦克位置随机生成，有一定概率不生成道具
3. **掉落概率**：==每种道具掉落概率均等（约 1/6），每次最多生成 1 个道具==
4. **闪烁特性**：所有道具生成后均会缓慢闪烁（亮→暗→亮），直到被玩家拾取或效果消失

### 2.2 交互规则

|交互类型|具体规则|视觉反馈|音效反馈|
|---|---|---|---|
|玩家拾取|道具接触玩家坦克瞬间消失，立即触发效果|道具消失，伴随短暂白色闪光|轻快的 "叮" 声（约 0.2 秒）|
|敌方接触|部分道具对敌方有特殊效果（如炸弹、秒表），部分无效果|敌方接触炸弹会立即爆炸；接触秒表无效果|敌方爆炸时播放重型爆炸音效；接触其他道具无音效|
|时间过期|秒表、头盔、铲子效果结束后自动失效|效果结束时伴随短暂灰色闪光|轻微的 "嗡" 声（约 0.1 秒）|

## UI设计核心特点

1. 信息层级与可读性
- 优先级明确:基地>玩家坦克>地方坦克>地形>道具，通过颜色、大小、位置强化视觉引导。
- 极简文字:仅保留关键信息，避免阅读负担，符合动作游戏快速决策需求
- 图标化设计:用坦克、老鹰等图标代替文字，降低语言门槛，==全球玩家都可以快速理解==。
2. 色彩系统
- 高对比度配色:黑色背景+白色文字/图标，战场元素色彩饱和度高(黄/绿/红/蓝)。
- 语义化色彩:不同等级敌方坦克用颜色区分(比如红色坦克为特殊型，会掉落道具)，增强游戏深度。
3. 交互设计
- 操作极简:仅用方向键+AB键，学习成本极低。
- 及时反馈:操作响应无延迟，炮弹发射/爆炸/坦克移动均有明确音效+动画，强化游戏感。
- 容错设计:坦克移动有碰撞检测，不会卡墙，炮弹有射程限制，避免无意义浪费。
- 动态难度提示:敌方坦克数量实时显示，帮助玩家判断战场压力。
